# Codero v1.2 Proving Gate — Runbook

## Overview

`codero prove` is the deterministic v1.2 proving entrypoint.  It runs all
high-signal checks in a single pass and emits both a human-readable summary table
and a stable JSON object.  It is the canonical command for CI evidence capture and
daily operator proving verification.

---

## Command Invocation

```bash
# Standard run (includes race tests — takes ~2 min)
codero prove

# Fast mode (skips C-004 race tests — ~15 seconds)
codero prove --fast

# JSON output only (suppress human table; JSON goes to stdout)
codero prove --json

# With running daemon (enables observability smoke checks C-007..C-009)
codero prove --endpoint-url http://localhost:8080

# Non-default repo root
codero prove --repo-path /path/to/codero/repo
```

### Exit codes

| Code | Meaning |
|---|---|
| `0` | All executed checks PASS (SKIP and TODO are not failures) |
| `1` | One or more checks returned FAIL |

---

## Output Schema

JSON is written to **stdout** (so it can be captured by CI).  Human table is
written to **stderr**.  Schema version is `"1"`; additions are additive only.

```json
{
  "schema_version": "1",
  "timestamp": "2026-03-17T00:57:41Z",
  "version": "v1.2.0",
  "repo_path": "/srv/storage/repo/codero",
  "overall_status": "PASS",
  "total": 22,
  "passed": 14,
  "failed": 0,
  "skipped": 4,
  "todo_count": 4,
  "failing_checks": [],
  "checks": [
    {
      "id": "C-001",
      "name": "go-build",
      "appendix_ref": "",
      "status": "PASS",
      "detail": "OK",
      "duration_ms": 1860
    }
  ]
}
```

### `status` values for individual checks

| Value | Meaning |
|---|---|
| `PASS` | Check executed and passed |
| `FAIL` | Check executed and failed |
| `SKIP` | Check not applicable (e.g. daemon not running for smoke check) |
| `TODO` | Known coverage gap — drill scaffolded, not yet implemented |

## Pass Criteria

A run is considered passing when:

- `overall_status == "PASS"` in the JSON output
- `failed == 0`
- `failing_checks` is empty (`[]`)
- SKIP checks (daemon not running, `--fast` race skip) are **not** failures
- TODO checks are **not** failures — they represent acknowledged coverage gaps

## Troubleshooting

### C-001 / C-002 / C-003 FAIL

Go toolchain check failed.

1. Verify Go version: `go version` — must be `go1.24.2` or compatible.
2. Run manually: `go build ./...` and inspect the error output in the `detail` field.
3. If missing modules: `go mod download && go mod tidy`.
4. Ensure you are running `codero prove` from the repo root (or pass `--repo-path`).

### C-007 / C-008 / C-009 show FAIL (not SKIP)

The daemon is running but returning a non-2xx/3xx response.

1. Check daemon logs: `codero status`
2. Run `codero dashboard --check` for detailed endpoint diagnostics.
3. Verify port binding: `codero ports`
4. If behind a reverse proxy, pass `--endpoint-url https://your-proxy-base`.

### C-019 / C-020 / C-021 / C-022 are TODO

These are known coverage gaps explicitly scaffolded as TODO checks.  They are
not failures — they represent Appendix G drills that require dedicated test
implementations.  See `cmd/codero/prove_cmd.go` for the `TODO(COD-031)` markers
to track progress.

## CI Integration

Capture JSON evidence in CI:

```bash
# Capture JSON to file; human summary goes to stderr (visible in CI logs)
codero prove --fast 2>prove-human.txt | tee prove-result.json
echo "Overall: $(jq -r .overall_status prove-result.json)"
```

Or fail the CI step on any FAIL:

```bash
codero prove --fast   # exits 1 if any FAIL, 0 if all PASS/SKIP/TODO
```
