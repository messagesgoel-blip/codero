## What Changed (vs v1.0.1)

### New CLI Commands

| Command | Purpose |
|---|---|
| `codero tui` | Canonical Bubble Tea interactive operator shell |
| `codero dashboard` | Print dashboard URL; `--check` validates 3 endpoints; `--open` opens browser |
| `codero ports` | Diagnostic table of all network bindings with conflict detection |

### `gate-status` Improvements

- `--json`: machine-readable JSON output, non-blocking, CI-safe
- `--no-prompt`: suppress interactive action prompt unconditionally
- TTY detection centralised via `tui.IsInteractiveTTY()` (was ad-hoc per call)
- Non-interactive exits: 1 on FAIL, 0 on PASS/PENDING (deterministic for scripts)

### Config Normalization (3 new optional fields)

| Field | Env override | Default |
|---|---|---|
| `observability_host` | `CODERO_OBSERVABILITY_HOST` | `""` (all interfaces) |
| `dashboard_base_path` | `CODERO_DASHBOARD_BASE_PATH` | `/dashboard` |
| `dashboard_public_base_url` | `CODERO_DASHBOARD_PUBLIC_BASE_URL` | `""` |

### Web Port / Routing Hardening

- `NewObservabilityServer` accepts explicit `host` and `dashboardBasePath` parameters
- SPA + API routes served under configurable base path (reverse-proxy safe)
- No hardcoded absolute paths in static asset references

### Internal

- `internal/tui/tty.go`: `IsInteractiveTTY()` via `os.ModeCharDevice` bitmask (no new deps)
- `internal/tui/app.go`: `Config.InitialTab` for `--view` flag wiring
- `internal/tui/theme.go`: `Theme.Name` string field for testability
