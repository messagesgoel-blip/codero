# Codero v1.1.0 — Operator User Manual

## What's New in v1.1.0

v1.1.0 adds four polished operator commands, makes `gate-status` CI-safe, and hardens the web
dashboard configuration for reverse-proxy and remote deployments.

**New commands**: `codero tui` · `codero dashboard` · `codero ports`  
**Improved**: `codero gate-status` (JSON output, non-interactive mode, deterministic exit codes)  
**Config**: three new optional fields for binding/base-path control (safe defaults; no migration needed)

## Install / Upgrade

### Prerequisites

- Go 1.24.2+ (for building from source)
- Shared tools binary target: `/srv/storage/shared/tools/bin/`

### Option A — Replace from shared release (recommended)

```bash
# Verify new binary
/srv/storage/shared/tools/releases/codero-v1.1.0/codero version   # → v1.1.0

# Atomic install
cp /srv/storage/shared/tools/releases/codero-v1.1.0/codero \
   /srv/storage/shared/tools/bin/codero.new
mv /srv/storage/shared/tools/bin/codero.new /srv/storage/shared/tools/bin/codero

# Confirm
codero version   # → v1.1.0
```

### Option B — Build from source

```bash
cd /srv/storage/repo/codero
git checkout main && git pull
go build -trimpath \
  -ldflags "-s -w -X main.version=v1.1.0" \
  -o /usr/local/bin/codero \
  ./cmd/codero
codero version   # → v1.1.0
```

### Rollback

```bash
cp /srv/storage/shared/tools/bin/codero.v1.0.1.bak \
   /srv/storage/shared/tools/bin/codero
codero version   # → v1.0.1
```

## First-Run Checks

After upgrading, run these to confirm a healthy installation:

```bash
codero version            # must print v1.1.0
codero status             # daemon state + redis status
codero ports              # all network bindings — no errors expected
codero gate-status --json # JSON output of current gate state
```

Start the daemon (if not already running):

```bash
codero start
codero dashboard --check  # validates /dashboard/, /api/v1/dashboard/overview, /gate
```

## Daily Operations

### Check gate status

```bash
# Human-readable (interactive terminal)
codero gate-status

# Machine-readable / CI
codero gate-status --json
# Example output:
# {"status":"PASS","repo":"/path/to/repo","timestamp":"2026-03-16T12:00:00Z","findings":0}

# Watch mode (live updates)
codero gate-status --watch --interval 3
```

### Open the dashboard

```bash
# Print URL and endpoint status
codero dashboard

# Validate all dashboard endpoints are reachable
codero dashboard --check   # exits 0 on all-OK, 1 on any failure

# Open in browser (interactive sessions only; skipped in CI/headless)
codero dashboard --open
```

### Inspect network bindings

```bash
codero ports
# Example output:
# SERVICE           BIND                           URL
# observability     0.0.0.0 (all interfaces):8080  http://localhost:8080
# dashboard SPA     0.0.0.0 (all interfaces):8080  http://localhost:8080/dashboard/
# gate endpoint     0.0.0.0 (all interfaces):8080  http://localhost:8080/gate
```

### Launch interactive TUI

```bash
codero tui                          # default: queue + gate overview
codero tui --view gate              # start on gate view
codero tui --view events            # start on events view
codero tui --interval 3             # refresh every 3 seconds
codero tui --theme dracula          # available: dark light system dracula vscode
codero tui --no-alt-screen          # for CI/screen capture environments
codero tui -r /path/to/repo         # target specific repo
```

## Configuration Reference

All new config fields are optional. Existing deployments require no changes.

Add to your `codero.yaml` (or pass as environment variables) only if you need non-default behaviour:

```yaml
# Bind address for the observability/dashboard server.
# Default: "" (binds to all interfaces, equivalent to 0.0.0.0)
# Use "127.0.0.1" to restrict to localhost only.
observability_host: ""

# URL prefix under which the dashboard SPA is served.
# Default: /dashboard
# Change this for reverse-proxy sub-path deployments.
dashboard_base_path: /dashboard

# Override the public base URL printed by `codero dashboard`.
# Useful behind a reverse proxy where localhost:8080 is not the public address.
# Default: "" (computed from observability_host + port + dashboard_base_path)
dashboard_public_base_url: https://ops.example.com/codero/dashboard
```

Environment variable equivalents:

| Variable | Config key |
|---|---|
| `CODERO_OBSERVABILITY_HOST` | `observability_host` |
| `CODERO_DASHBOARD_BASE_PATH` | `dashboard_base_path` |
| `CODERO_DASHBOARD_PUBLIC_BASE_URL` | `dashboard_public_base_url` |

## Troubleshooting FAQ

**`codero dashboard --check` returns "unreachable"**

The observability daemon is not running. Start it with `codero start`, then re-run the check.

**`codero ports` shows webhook as "(disabled; polling-only mode)"**

This is expected when `webhook.enabled` is false in config. No action needed unless you want
push-based webhook delivery.

**`codero gate-status --json` exits 1**

The current gate state is FAIL. Inspect the findings:
```bash
codero gate-status --json | jq .
codero gate-status --logs   # detailed log output
```

**`codero tui` says "requires an interactive terminal"**

You are running in a non-TTY context (pipe, CI, script). Use `codero gate-status` for non-interactive output instead.

**Dashboard base path returning 404 behind reverse proxy**

Ensure your proxy strips or preserves the path prefix consistently with `dashboard_base_path`.
Example (nginx):
```nginx
location /codero/dashboard/ {
    proxy_pass http://localhost:8080/dashboard/;
}
```
Or set `dashboard_base_path: /codero/dashboard` to match the proxy path without rewriting.

**`codero version` still shows v1.0.1 after upgrade**

The shell may be caching the old binary path. Run `hash -r` (bash) or open a new terminal, then retry.

## Recovery / Rollback Quick Steps

```bash
# 1. Stop daemon if running
codero stop

# 2. Restore previous binary
cp /srv/storage/shared/tools/bin/codero.v1.0.1.bak \
   /srv/storage/shared/tools/bin/codero

# 3. Confirm rollback
codero version   # → v1.0.1

# 4. Restart
codero start
codero status
```

## Command Reference

### `codero tui`

Launch the interactive Bubble Tea operator shell.

| Flag | Default | Description |
|---|---|---|
| `-r, --repo-path` | current dir | Target repo path |
| `--interval` | `5` | Refresh interval in seconds |
| `--theme` | `system` | `dark` · `light` · `system` · `dracula` · `vscode` |
| `--view` | `gate` | Initial tab: `gate` · `queue` · `events` · `output` · `findings` |
| `--no-alt-screen` | false | Disable alternate screen buffer |

### `codero gate-status`

One-shot or watch gate status.

| Flag | Description |
|---|---|
| `--watch` | Live watch mode |
| `--interval N` | Watch refresh interval (seconds) |
| `--json` | JSON output, non-interactive, CI-safe |
| `--no-prompt` | Suppress action prompt even in TTY |
| `--logs` | Show gate log tail |

### `codero dashboard`

Print dashboard URL and optionally validate endpoints.

| Flag | Description |
|---|---|
| `--host` | Override host (default from config) |
| `--port` | Override port (default from config) |
| `--check` | Validate dashboard SPA, overview API, and /gate; exit 1 on any failure |
| `--open` | Open dashboard in browser (interactive sessions only) |

### `codero ports`

Print all effective network bindings. Warns on port conflicts.

### `codero version`

Print the installed binary version.

### `codero status`

Print daemon, redis, and service state.

### `codero start` / `codero stop`

Start or stop the background daemon.
