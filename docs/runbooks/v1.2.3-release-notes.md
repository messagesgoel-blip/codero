## What's New

### BUG-001 Fix: forbidden-paths disabled reason message (COD-052)

**Problem:** When `CODERO_ENFORCE_FORBIDDEN_PATHS=1` was set but `CODERO_FORBIDDEN_PATH_REGEX`
was left empty, the gate engine reported `reason: "CODERO_ENFORCE_FORBIDDEN_PATHS not set"` —
which is factually incorrect. The enforce flag was set; the regex was missing.

**Fix:** The disabled reason now distinguishes the two sub-conditions:
- `CODERO_ENFORCE_FORBIDDEN_PATHS not set` — when the enforce flag is false (unchanged)
- `CODERO_FORBIDDEN_PATH_REGEX not set or empty` — when enforce is true but regex is empty (new)

**Impact:** Operators who set `CODERO_ENFORCE_FORBIDDEN_PATHS=1` and forget
`CODERO_FORBIDDEN_PATH_REGEX` now receive an actionable, accurate error message.

**Files changed:**
- `internal/gatecheck/engine.go` — fix disabled reason message
- `internal/gatecheck/engine_test.go` — add BUG-001 regression test; assert reason in existing test
- `docs/contracts/gate-check-env-contract-v1.md` — document two-var requirement

## Migration Notes

### forbidden-paths check activation
To enable the forbidden-paths gate check, **both** variables must be set:

```bash
export CODERO_ENFORCE_FORBIDDEN_PATHS=1
export CODERO_FORBIDDEN_PATH_REGEX='\.env$|secrets/'
```

Setting only `CODERO_ENFORCE_FORBIDDEN_PATHS=1` without a regex leaves the check disabled.
v1.2.3 now reports a clear reason message in this case.
