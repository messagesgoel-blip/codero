# Codero v1.2.0 — Operator User Manual

## What's New in v1.2.0

### `codero prove` — Proving Gate Command

The headline feature of v1.2.0 is the proving gate: a single command that runs all
22 high-signal checks and emits both a human-readable table and a stable JSON evidence file.

```bash
codero prove --fast --repo-path /path/to/codero
codero prove --json --repo-path /path/to/codero   # JSON to stdout
```

Use `--fast` to skip the race-test pass (faster for daily use). Drop it for a full pre-release run.

Key behavior:

- **Exit 0** when all checks PASS or SKIP. **Exit 1** on any FAIL.
- **SKIP** (not FAIL) for daemon-dependent checks when the daemon is not running.
- **JSON schema** is stable: `schema_version: "1"` with explicit `pass`/`fail`/`skip` per check.

### Non-TTY `gate-status --watch`

`gate-status --watch` no longer errors in CI, Docker, or piped shells. When stdout/stdin are not
TTYs, it emits a single JSON object (same schema as `--json`) and exits 0.

### GitHub Review Integration

The `codero` daemon now uses the real GitHub API for review events. The `/gate/why` endpoint
reflects live PR review data.

### CI Proving Gate Workflow

Every PR to `main` and every push to `main` automatically runs the proving gate in GitHub Actions.
Artifacts (human output + JSON) are retained for 30 days per run.

## Install / Upgrade Instructions

### First install

```bash
# Build from source
cd /srv/storage/repo/codero
git checkout main && git pull

go build -trimpath \
  -ldflags "-s -w -X main.version=v1.2.0" \
  -o /srv/storage/shared/tools/bin/codero \
  ./cmd/codero

# Verify
codero version   # → v1.2.0
```

### Upgrade from v1.1.0

```bash
# 1. Back up current binary
cp -p /srv/storage/shared/tools/bin/codero \
      /srv/storage/shared/tools/bin/codero.v1.1.0.bak

# 2. Build new binary
cd /srv/storage/repo/codero
git checkout main && git pull

go build -trimpath \
  -ldflags "-s -w -X main.version=v1.2.0" \
  -o /srv/storage/shared/tools/releases/codero-v1.2.0/codero \
  ./cmd/codero

# 3. Atomic swap
cp /srv/storage/shared/tools/releases/codero-v1.2.0/codero \
   /srv/storage/shared/tools/bin/codero.new
mv /srv/storage/shared/tools/bin/codero.new \
   /srv/storage/shared/tools/bin/codero

# 4. Verify
codero version                                         # → v1.2.0
codero prove --fast --repo-path /srv/storage/repo/codero   # → overall=PASS
```

> **PATH note:** `codero` is available via `~/.local/bin/codero` (symlink to the shared tools
> binary). If `codero` is not found after upgrade, run:
> `ln -sf /srv/storage/shared/tools/bin/codero ~/.local/bin/codero`

## First-Run Checks

After install or upgrade, confirm health with:

```bash
codero version                                     # must print v1.2.0
codero status                                      # shows gate state (daemon may be off — OK)
codero gate-status --json                          # valid JSON with status/run_id/comments keys
codero ports                                       # shows port binding table
codero prove --fast --repo-path /path/to/codero    # overall=PASS, 0 FAIL
```

Expected output summary from `prove --fast`:

```
C-001  go-build                               PASS
C-002  go-vet                                 PASS
C-003  unit-tests (20 pkgs)                   PASS
C-004  race-tests                             SKIP  (--fast)
...
C-022  drill:G-012 compaction                 PASS

overall=PASS  passed=18  failed=0  skipped=4  todo=0
```

## Daily Operations

### Check gate status

```bash
codero gate-status             # human-readable TUI (TTY)
codero gate-status --json      # machine-readable (TTY or CI)
codero gate-status --watch     # auto-refresh TUI (TTY); JSON snapshot (non-TTY)
```

### Run proving gate

```bash
# Quick (daily dev check — skips race tests)
codero prove --fast --repo-path /path/to/codero

# Full (pre-release or post-incident)
codero prove --repo-path /path/to/codero

# JSON evidence (for scripting)
codero prove --json --repo-path /path/to/codero | jq .
```

### Dashboard

```bash
codero dashboard --check       # verify all 3 observability endpoints
codero dashboard               # open dashboard URL (requires daemon running)
```

### Ports and process status

```bash
codero ports                   # all known port bindings
codero status                  # current lifecycle state
```

## Troubleshooting FAQ

**Q: `codero prove` exits 1 — which check failed?**

Read the human output (stderr) or parse the JSON:

```bash
codero prove --json --repo-path /path/to/codero \
  | jq '.checks[] | select(.status == "fail")'
```

---

**Q: C-001/C-002/C-003 fail with "directory prefix . does not contain main module"**

You ran `codero prove` from a directory other than the codero repo. Always pass `--repo-path`:

```bash
codero prove --fast --repo-path /srv/storage/repo/codero
```

---

**Q: C-007/C-008/C-009 report SKIP**

The daemon is not running. These checks require a live HTTP listener on the observability ports.
Start the daemon and re-run, or accept SKIP as expected in dev/CI environments.

---

**Q: `gate-status --watch` hangs or errors in CI**

Since v1.2.0 this should not occur. Verify you are running the v1.2.0 binary:

```bash
codero version   # must be v1.2.0
```

If on an older binary, upgrade using the steps in **Upgrade from v1.1.0** above.

---

**Q: `codero prove --json` output has `schema_version != "1"`**

You may be running a pre-release or dev build. Rebuild from the `main` branch at commit `826b2fc`
or later. The CI workflow enforces this check automatically.

---

**Q: The CI `proving-gate` workflow fails on JSON sanity**

1. Download the artifact from the GitHub Actions run.
2. Inspect `prove-result.json` — look for `"status":"fail"` entries.
3. Run `codero prove --json --repo-path .` locally and reproduce.
4. Fix the failing check and push. The workflow re-runs on the new push.

## Command Reference

| Command | Description |
|---|---|
| `codero version` | Print installed version |
| `codero status` | Current lifecycle/gate state |
| `codero gate-status` | TUI gate status (TTY) / JSON snapshot (non-TTY) |
| `codero gate-status --json` | Gate status as JSON (always, any terminal) |
| `codero gate-status --watch` | Auto-refresh (TUI in TTY; JSON once in non-TTY) |
| `codero prove --fast` | Run all checks, skip race tests |
| `codero prove` | Full check suite including race tests |
| `codero prove --json` | Emit JSON evidence to stdout (human table to stderr) |
| `codero dashboard` | Open dashboard URL |
| `codero dashboard --check` | Verify observability endpoints reachable |
| `codero ports` | Show port binding table |
| `codero start` | Start the Codero daemon |
| `codero stop` | Stop the Codero daemon |

### `codero prove` flags

| Flag | Default | Description |
|---|---|---|
| `--fast` | false | Skip race-test pass (C-004). Faster daily-dev mode. |
| `--json` | false | Emit JSON evidence to stdout; human table goes to stderr. |
| `--repo-path <dir>` | `.` (cwd) | Path to the codero repo root. Required if cwd ≠ repo root. |
