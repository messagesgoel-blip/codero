## Phase 2 — Release build

The v1.2.0 artifact previously stored at
`/srv/storage/shared/tools/releases/codero-v1.2.0/` was built before COD-036
merged and did not include the PID cleanup fix. The artifact was rebuilt as
part of this drill to represent current `main` HEAD.

```bash
VERSION=v1.2.0
RELEASE_DIR=/srv/storage/shared/tools/releases/codero-${VERSION}

go build -trimpath \
  -ldflags "-s -w -X main.version=${VERSION}" \
  -o "${RELEASE_DIR}/codero" \
  ./cmd/codero

sha256sum "${RELEASE_DIR}/codero" | tee "${RELEASE_DIR}/codero.sha256"
```

## Release tagging guidance (final promotion)

When all staging checks are complete, tag and promote with:

```bash
# From repo root on main (after merging release PR)
git checkout main && git pull

# Tag
git tag -a v1.2.0 -m "Release v1.2.0 — COD-031..COD-036"
git push origin v1.2.0

# Atomic install swap
VERSION=v1.2.0
RELEASE_DIR=/srv/storage/shared/tools/releases/codero-${VERSION}
INSTALL_TARGET=/srv/storage/shared/tools/bin/codero

cp -p "${INSTALL_TARGET}" "${INSTALL_TARGET}.prev.bak"
cp "${RELEASE_DIR}/codero" "${INSTALL_TARGET}.new"
mv "${INSTALL_TARGET}.new" "${INSTALL_TARGET}"

# Verify
codero version        # → v1.2.0
codero status         # → not running (or running if daemon was live)

# Run full verify-release with live daemon (if running)
CODERO_BIN="${INSTALL_TARGET}" bash scripts/automation/verify-release.sh \
  --version v1.2.0 \
  --repo-path /srv/storage/repo/codero \
  --endpoint-url http://localhost:8080
```
