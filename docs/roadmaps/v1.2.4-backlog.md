## Backlog Items

### UI-001: TUI Visual Design Refresh (P1)
**Surface:** TUI (Bubble Tea)  
**Owner:** ClaudeCode / UI agent  
**Description:** Modernize the Bubble Tea layout for the `codero tui` and `codero gate-status`
views. Current layout is functional but visually dense.

**Acceptance criteria:**
- Gate view uses clear status icons with colour-coding (pass=green, fail=red, skip=yellow, disabled=grey)
- Summary line is visually distinct from the per-check table
- Layout degrades gracefully in narrow terminals (< 80 cols)
- `--tui-snapshot` output remains deterministic and parseable (no ANSI escape sequences)
- All existing TUI/JSON parity tests pass

**Pilot reference:** P3 + P5 evidence from Pilot Rerun Batch 2

### INFRA-001: Distinguish `/gate` (heartbeat) from `/gate-check` (local checks) surface naming (P2)
**Surface:** Dashboard API  
**Owner:** OpenCode / API agent  
**Description:** The `/gate` endpoint returns AI review gate heartbeat status; the
`/api/v1/dashboard/gate-checks` endpoint returns local pre-commit gate-check results.
These coexist correctly but their naming is non-obvious to new operators.

**Acceptance criteria:**
- API contract documentation updated to clarify both endpoints with usage examples
- Optional: add a `/api/v1/dashboard/gate-check` alias endpoint for clarity
- No breaking changes to existing consumers
